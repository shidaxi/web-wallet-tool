# 助记词 & 多链钱包生成器 —— 纯静态 Web 工具

## 项目定位
纯前端静态页面（React + 无后端），所有密钥计算在浏览器本地完成，
零网络请求，零数据上传，可离线使用。面向开发者/研究者的专业工具。

---

## 技术栈约束
- React（单文件或 Vite 构建均可）
- 纯客户端计算，禁止任何网络请求
- 依赖库（CDN 或 npm）：
  - `bip39` — 助记词生成与校验
  - `@scure/bip32` 或 `hdkey` — HD 派生
  - `@solana/web3.js` — Solana 地址
  - `@mysten/sui.js` — Sui 地址
  - `bitcoinjs-lib` — BTC 地址
  - `ethers.js` v6 — EVM 系地址
  - `@noble/ed25519` / `@noble/secp256k1` — 底层签名
  - `libp2p` 或 `@libp2p/peer-id-factory` — P2P Node ID 计算
- 样式：Tailwind CSS 或手写 CSS，极简深色主题

## 功能模块详细说明

### 模块一：助记词生成器

**生成控制**
- 支持选择助记词字数：12 / 15 / 18 / 21 / 24 词
- 支持选择助记词语言：英文（BIP39 标准）/ 中文简体
- 一键随机生成，按钮带刷新图标
- 可输入自定义 Passphrase（BIP39 第 25 个词，默认为空）

**助记词展示与编辑**
- 每个单词显示为独立卡片，卡片左上角显示序号（1–24）
- 每张卡片包含一个下拉选择框（searchable dropdown）：
  - 下拉列表包含完整 BIP39 词表（2048 个词）
  - 支持输入筛选，模糊匹配
  - 选中词高亮显示
  - 修改任意单词后，自动重新校验助记词合法性
- 助记词合法性状态指示：
  - 合法：绿色边框 + "✓ Valid Mnemonic"
  - 非法：红色边框 + "✗ Invalid Checksum"

**导入功能**
- 支持粘贴已有助记词（空格分隔）自动解析填充
- 解析后自动校验合法性

### 模块二：HD Path 配置

提供预设 + 自定义两种模式：

**预设 HD Path 列表**
| 链 / 标准 | HD Path |
|-----------|---------|
| Ethereum (BIP44) | m/44'/60'/0'/0 |
| Solana (BIP44) | m/44'/501'/0'/0' |
| Sui (BIP44) | m/44'/784'/0'/0'/0' |
| Bitcoin Legacy (BIP44) | m/44'/0'/0'/0 |
| Bitcoin SegWit (BIP49) | m/49'/0'/0'/0 |
| Bitcoin Native SegWit (BIP84) | m/84'/0'/0'/0 |
| Bitcoin Taproot (BIP86) | m/86'/0'/0'/0 |
| Cosmos / ATOM | m/44'/118'/0'/0 |
| Aptos | m/44'/637'/0'/0'/0' |
| EVM 通用 | m/44'/60'/0'/0 |

**自定义模式**
- 支持手动输入任意 HD Path 字符串
- 实时格式校验，非法格式红色提示

---

### 模块三：链类型选择

单选或多选目标链，当前支持：

| 分类 | 支持链 |
|------|--------|
| EVM 系 | Ethereum / BNB Chain / Polygon / Avalanche / Arbitrum |
| Solana | Solana |
| Move 系 | Sui / Aptos |
| UTXO 系 | Bitcoin (Legacy / SegWit / Native SegWit / Taproot) |
| Cosmos 系 | Cosmos (ATOM) |

选择不同链时，HD Path 预设自动联动切换为推荐默认值，用户仍可手动覆盖。

---

### 模块四：派生钱包列表

**显示控制**
- 默认展示前 5 个派生账户（索引 0–4）
- 支持调整显示数量：5 / 10 / 20 / 50
- 支持手动输入起始索引（如从第 10 个开始派生）

**每个钱包条目展示字段**
| 字段 | 说明 |
|------|------|
| # 序号 | 派生索引 |
| HD Path | 完整路径（如 m/44'/60'/0'/0/0） |
| 地址 | 链格式地址（带链前缀图标） |
| 公钥 | 压缩公钥 hex 或 base58 |
| 私钥 | 默认隐藏，点击眼睛图标显示/隐藏，hex 或 base58 格式 |

**复制功能**
- 每个字段右侧配有复制图标
- 单击即复制到剪贴板
- 复制后图标短暂变为 ✓ 确认状态（1.5 秒后恢复）

**批量导出**
- 支持将当前所有派生钱包数据导出为 JSON 文件
- 导出前弹出安全警告确认框

---

### 模块五：P2P Node Key 工具

面向使用私钥作为 P2P 网络节点身份的场景（如 libp2p、substrate 等）。

**输入方式**
- 方式 A：从上方派生钱包列表中选择一个私钥（点击"用作 Node Key"按钮）
- 方式 B：直接粘贴输入任意私钥（hex 格式）

**计算输出**
| 字段 | 说明 |
|------|------|
| 私钥 (hex) | 输入的原始私钥 |
| 公钥 (hex) | 对应 ed25519 或 secp256k1 公钥 |
| PeerId (base58) | libp2p 标准 PeerId，格式：`12D3Koo...` |
| PeerId (base58btc multihash) | 完整 multihash 格式 |
| Node Key 文件内容 | 可直接用于 substrate / libp2p 节点配置的格式 |

**曲线选择**
- Ed25519（默认，Substrate / libp2p 常用）
- Secp256k1（以太坊 / BTC 系节点）

**一键复制** 每个输出字段均支持复制。

## UI / UX 设计规范

**整体风格**
- 深色主题（背景 #0f1117，卡片 #1a1d27）
- 主色调：靛蓝 #6366f1 或青绿 #00d4aa（点缀按钮/高亮）
- 字体：Monospace（JetBrains Mono / Fira Code）用于密钥展示，
  无衬线（Inter / system-ui）用于 UI 标签

**布局结构**
```
┌─────────────────────────────────────────┐
│  顶部 Header：工具名称 + 离线安全提示徽章  │
├─────────────────────────────────────────┤
│  Section 1：助记词生成器                  │
│    [字数选择] [语言] [生成按钮]            │
│    [单词卡片 × N]                        │
│    [Passphrase 输入] [有效性状态]         │
├─────────────────────────────────────────┤
│  Section 2：配置面板（横向三列）           │
│    [链选择] | [HD Path] | [显示数量]      │
├─────────────────────────────────────────┤
│  Section 3：派生钱包列表（表格）           │
│    [#][路径][地址][公钥][私钥][操作]       │
│    [加载更多 / 调整数量]                  │
├─────────────────────────────────────────┤
│  Section 4：P2P Node Key 工具（可折叠）   │
└─────────────────────────────────────────┘
```

**安全提示**
- 页面顶部常驻黄色警告横幅：
  "⚠ 此工具仅在本地运行，请勿在联网环境下使用真实资产的助记词"
- 私钥字段默认遮盖（blur 效果），需主动点击才显示
- 导出 JSON 前必须二次确认

**交互细节**
- 助记词单词卡片支持拖拽排序（用于验证记忆顺序）
- 所有密钥字段 hover 时显示复制按钮
- 表格行 hover 轻微高亮
- 响应式布局，支持宽屏和移动端

## 安全约束（硬性要求）

1. **禁止任何 fetch / axios / XHR 网络请求**
2. **禁止 localStorage / sessionStorage 持久化任何密钥数据**
3. 页面关闭后内存中所有数据自动清除
4. 纯静态文件，可直接双击 index.html 离线运行，或托管于任意静态服务器
